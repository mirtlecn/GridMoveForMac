# GridMove UI / UX 说明

本文档定义 GridMove 的 UI / UX 的设计逻辑、界面结构、和交互行为。

软件底层的事件流请参考 `APP-DESIGN.md`。

## 1. UI 界面概览

GridMove UI 由三个界面组成：

- [菜单栏菜单界面](#4-菜单栏-ui--ux-设计)
- [辅助功能权限请求](#5-辅助功能权限请求)
- [设置窗口](#6-设置界面-ui--ux-设计)（2026-04-17 注：暂未实现，用菜单栏的 Customize... 和 Reload 入口暂时替代）

## 2. UI 界面基本规范

UI 尽量使用原生控件和标准窗口行为，除非现有 UI 已经有需要，否则避免引入自定义交互模型。

可见的 UI 文案应遵循以下规则：

- 默认使用 sentence case
- 只有首单词首字母大写
- 缩写和专有名词按实际需要保留大写，例如 `Bundle ID`、`GridMove`

当前可见 UI 文案统一集中在：

- `Sources/GridMove/Support/UICopy.swift`

该文件用于：

- 统一菜单栏文案
- 统一权限引导窗口文案
- 统一设置窗口文案
- 为后续翻译提供单一入口

## 4. 菜单栏 UI / UX 设计

菜单栏菜单：

- `Enable`
- 分隔线
- `Middle mouse drag`
- `Modifier + left mouse drag`
- `Prefer layout mode`
- 分隔线
- 布局操作
- 分隔线
- `Reload`
- `Customize... ↗`
- `Quit`

如后续实现了设置窗口，`Customize...` 和 `Reload` 将被替换为 `Settings...`，点击后打开设置窗口。

交互：

- `Enable`：是全局运行时开关
- `Middle mouse drag`：控制是否启用鼠标中键拖动触发。菜单项显示当前开关状态，点击后立即生效，并写回配置文件。
- `Modifier + left mouse drag`：控制是否启用修饰键 + 鼠标左键拖动触发。菜单项显示当前开关状态，点击后立即生效，并写回配置文件。
- `Prefer layout mode`：控制进入拖动触发态后默认进入布局模式还是移动模式。菜单项显示当前开关状态，点击后立即生效，并写回配置文件。
- 布局操作：
    - 文案行为从当前配置生成，固定文案为 Apply <layout name>，如果没有 Name，则文案改为 Apply <layout id>
    - 文案后，展示对应的设置的第一个快捷键（如有）
    - 点击后，对焦点窗口执行对应行为
- 菜单项应触发现有操作路径，而非维护自身状态
- `Reload`: 从磁盘重新加载配置文件
- `Customize... ↗`: 在 Finder 中打开配置目录
- `Quit`: 退出应用

## 5. 辅助功能权限请求

当前没有额外的辅助功能引导窗口。

当前行为：

- 应用启动时，如果缺少辅助功能权限，直接触发一次系统辅助功能权限请求
- 如果用户拒绝，本次运行不再重复请求
- 如果应用运行中权限从已授权变为未授权，会再次触发一次系统请求
- 如果用户再次拒绝，本次运行继续保持不可用，直到下次启动再请求

权限未授予时：

- 拖动触发停止工作
- 快捷键监听停止工作
- 应用仍保留在菜单栏中，不强制阻塞其它操作

### 6.1 设置界面概述

设置界面由两部分构成，第一部分为全局的设置切换入口，第二部分为每个 tab 内的设置项列表。

全局入口当前有五个
- `General`: 一些总体的全局设置
- `Layouts`: 布局概览、预览和编辑界面
- `Appearance`: 外观设置
- `Hotkeys`: 快捷键设置
- `About`: 关于信息

规则：
- 每个 tab 的顶部应当包含当前项的标题
- 有一个全局的前进后退按钮（如果有历史记录的话）。
- 点击设置项后，进入对应的设置详情界面。
- 除特殊说明，修改设置后，立即生效

### 6.2 General tab

General 设置项按顺序排列如下，如有新增项，按逻辑添加：

1. `Enable`: 是|否：控制整个应用的启用状态。
2. `Prefer layout mode`：是|否：控制默认进入布局选择模式还是移动模式。
2. 此处使用样式分割
3. `Trigger method`: 此设置组控制布局触发行为：
    - `Middle mouse`：是否启用鼠标中键触发布局行为，默认是。
    - `Modifier + left mouse`：是否启用修饰键 + 鼠标左键触发布局行为，默认是。其下有一个附属列表，用于维护修饰键组合。
        - 修饰键组合列表，支持添加和删除组合。每个组合由一个或多个修饰键组成，例如 `Control + Option`、`Shift` 等。
        - 添加组合，用户可以选择一个或多个修饰键来创建新的组合。
        - 删除组合，用户可以选择已有的组合进行删除。无须确认
4. 此处使用样式分割
5. `Excluded windows`: 展示当前排除的窗口。每一项应当展示排除的类型（Bundle ID 或 Window title）和对应的值。可以删除和添加条目。可以参考修饰键组合列表的交互设计。

### 6.3 Layouts

Layouts tab 包含布局的概览、预览和编辑功能。包含两部分。
- 第一部分展示所有布局，必须展示布局的名称或者 ID（没有名称时），用户在这个部分可以调整布局的顺序，顺序会影响循环布局时的行为，用户可以添加或者删除布局。
- 第二部分展示某个布局的详情。

新增布局时，直接创建一个布局，名称留空。直接进入布局详情界面。

布局详情包含以下设置项：
1. `Name`: 布局名称，非必填，如果没有名称。
2. `Include in cycle`: 是否包含在循环布局中，默认为 true。
3. 样式分割
4. `Columns`: x `rows`: 默认 12x6。用户可以修改为正整数。此设置影响布局预览的网格划分。
5. 展示展示一个页内的 tab 切换，`Window layout` 和 `Trigger regions`，点击展示对应的布局预览。
    - 布局预览：使用一个共享的可视化面板组件。详情参考 [布局预览](#7-布局预览组件)。即使用户在 Appearance 关闭了对应的渲染，此处也应当展示渲染。用户可以在预览中通过拖动、选中、缩放等方式选择方块。
    - 切换到 Window layout 时，只展示窗口布局预览
    - 切换到 Trigger regions 时，展示触发区域预览
6. `Save`:布局预览修改非实时生效，必须用户显式点击此保存按钮。当修改了布局详情时，可以点击。此按钮应当和主设置部分位置有显著区别。例如放置到窗口右下角。点击后保存修改

### 6.4 Appearance

1. 布局预览组件：使用布局预览组件。此处完全依照用户设置展示布局预览，不可操作方块，仅作实时展示作用。
2. `Window layout` 设置组。控制窗口布局，窗口样式等。
    - `Render window overlay`: 是否渲染窗口布局的 Overlay
    - `Window layout gap`：布局中间的空隙。
    - `Fill opacity`：窗口布局中心的填充透明度，0 ~ 1
    - `Stroke width`：窗口布局描边的宽度
    - `Overlay color`
3. `Trigger area` 设置组，控制触发区域的展示样式
    - `Render trigger overlay`: 是否渲染 overlay
    - `Trigger area gap`: 空隙
    - `Overlay color`：颜色

### 6.5 Hotkeys

此页面展示动作和对应的快捷键。点击快捷键，可以录制新的快捷。

一个动作可以绑定多个快捷键。

动作文案等同菜单栏右键的文案： 
- Apply next layout / previous layout
- Apply <layout-name | layout-id>

如果快捷键为空， 动作不绑定快捷键。

### 6.6 About

此页面当前包含以下内容：

- Version: 展示当前版本号

后续可能会拓展检查更新、更新日志等功能。

## 7. 布局预览组件

多个页面复用同一套预览组件逻辑，目前包括：

- `Layouts` 详情页
- `Appearance` 页面

共享预览组件的逻辑要求：

- 使用统一的数据结构描述窗口区域和触发区域
- 支持屏幕区域和菜单栏区域。
- 菜单栏区域与屏幕区域共用同一套命中和显示逻辑
- 预览组件在不同页面中应保持同一套行为

菜单栏渲染：右侧渲染一个 12:00 的时间，左侧渲染一个 Apple log。

预览组件展示的内容由上层页面决定：

- `Layouts` 的窗口区域编辑，只显示窗口区域，可操作方块
- `Layouts` 的触发区域编辑，只显示触发区域，可操作方块
- `Appearance` 的预览，可同时显示窗口区域和触发区域
