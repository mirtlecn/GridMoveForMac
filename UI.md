# GridMove UI Logic

本文档记录当前主要 UI 的功能逻辑、页面结构、分组关系、状态依赖和交互行为。

本文档不记录视觉样式、间距、字号、最小尺寸等表现层细节。

## 1. UI 范围

当前 UI 范围包括：

- 菜单栏菜单
- 权限引导窗口
- 设置窗口

## 2. 文案规则

当前可见 UI 文案遵循以下规则：

- 默认使用 sentence case
- 只有首单词首字母大写
- 缩写和专有名词按实际需要保留大写，例如 `Bundle ID`、`GridMove`

当前可见 UI 文案统一集中在：

- `Sources/GridMove/Support/UICopy.swift`

该文件用于：

- 统一菜单栏文案
- 统一权限引导窗口文案
- 统一设置窗口文案
- 为后续翻译提供单一入口

## 3. 菜单栏菜单

菜单栏菜单包含以下固定结构：

- `Enable`
- 分隔线
- 布局动作项
- 分隔线
- `Settings…`
- 分隔线
- `Quit`

### 3.1 Enable

`Enable` 是全局开关的菜单入口。

点击后：

- 切换应用启用状态
- 同步更新设置页中的全局启用状态
- 同步影响拖动触发、快捷键和 CLI 动作

### 3.2 布局动作项

布局动作项由当前配置动态生成，顺序为：

- `Apply previous layout`
- `Apply next layout`
- 每个 layout 一项 `Apply <layout name>`

行为规则：

- 如果该动作配置了快捷键，菜单右侧显示第一个已启用快捷键
- 点击菜单项时，操作当前解析到的目标窗口
- 菜单项本身不维护独立的布局状态，只触发现有布局应用逻辑

### 3.3 Settings

点击 `Settings…` 后：

- 打开设置窗口
- 如果窗口已存在，则激活并显示已有窗口

### 3.4 Quit

点击 `Quit` 后：

- 退出应用

## 4. 权限引导窗口

当应用检测到缺少辅助功能权限时，会显示权限引导窗口。

当前窗口行为：

- 激活应用
- 显示引导窗口
- 不使用全局置顶层级
- 不强制压过系统设置自己的弹窗

窗口包含：

- 标题 `Accessibility access is required`
- 说明文案
- `Request accessibility access`
- `Open accessibility settings`

### 4.1 Request accessibility access

点击后：

- 触发系统辅助功能权限请求

### 4.2 Open accessibility settings

点击后：

- 打开系统设置中的辅助功能权限页面

## 5. 设置窗口总体结构

设置界面由两部分组成：

- 左侧侧边栏
- 右侧详情区域

侧边栏用于切换页面。

右侧详情区域由两层组成：

- 顶部固定标题栏
- 下方页面内容

页面切换后：

- 顶部标题栏保持存在
- 下方页面内容切换

## 6. 页面列表

当前设置界面包含 5 个页面：

- `General`
- `Layouts`
- `Appearance`
- `Hotkeys`
- `About`

点击侧边栏条目后：

- 当前页面切换到对应页面
- 写入页面导航历史

## 7. 顶部标题栏逻辑

顶部标题栏固定在详情区域顶部，不参与页面内容滚动。

标题栏包含：

- 返回
- 前进
- 当前标题

### 7.1 页面历史

除 `Layouts` 页内部的详情态外，前进和后退用于页面级历史。

页面级历史规则：

- 从一个页面进入另一个页面时，写入后退历史
- 后退会回到之前访问过的页面
- 前进会回到后退前所在页面

### 7.2 Layouts 页面内部导航

`Layouts` 页除了页面级历史，还包含自己的内部状态：

- 列表页
- 详情页

当 `Layouts` 处于详情页时：

- 顶部标题显示当前 layout 的显示编号
- 返回按钮优先返回 `Layouts` 列表页
- 从详情页返回到列表页后，前进按钮可以重新进入刚才那个详情页

## 8. 共享预览组件逻辑

多个页面复用同一套预览组件逻辑，目前包括：

- `Layouts` 详情页
- `Appearance` 页面

共享预览组件的逻辑要求：

- 使用统一的数据结构描述窗口区域和触发区域
- 支持屏幕区域和菜单栏区域
- 菜单栏区域与屏幕区域共用同一套命中和显示逻辑
- 预览组件在不同页面中应保持同一套行为

预览组件展示的内容由上层页面决定：

- `Layouts` 的窗口区域编辑，只显示窗口区域
- `Layouts` 的触发区域编辑，只显示触发区域
- `Appearance` 的预览，可同时显示窗口区域和触发区域

## 9. General 页面

`General` 页面包含 3 组逻辑：

- 总开关
- 鼠标触发设置
- 排除窗口设置

### 9.1 总开关

页面顶部是全局开关：

- `Enable`

它控制整个应用的启用状态。

关闭后：

- 拖动触发不可用
- 快捷键不可用
- CLI 动作不可用

### 9.2 鼠标触发设置

此组包含两个独立开关：

- `Middle mouse`
- `Modifier + left mouse`

这两项在逻辑上属于同一组。

当总开关关闭时：

- 整组禁用

#### 9.2.1 Middle mouse

控制中键长按是否可用于触发拖动网格。

#### 9.2.2 Modifier + left mouse

控制修饰键加左键是否可用于触发拖动网格。

其下有一个附属列表，用于维护修饰键组合。

这个附属列表与 `Modifier + left mouse` 有从属关系：

- 当 `Modifier + left mouse` 关闭时，该列表禁用
- 当 `Modifier + left mouse` 开启时，该列表可编辑

修饰键组合列表支持：

- 选择一项
- 添加
- 删除

添加操作打开弹窗。

删除操作：

- 只对当前选中项生效
- 不需要确认

当没有选中项时：

- 删除入口不显示

### 9.3 排除窗口设置

此组用于维护排除规则。

每条规则有两个字段：

- 值
- 类型

类型支持：

- `Bundle ID`
- `Window title`

排除规则列表支持：

- 选择一项
- 添加
- 删除

添加操作打开弹窗，让用户选择类型并输入值。

删除操作：

- 只对当前选中项生效
- 不需要确认

当没有选中项时：

- 删除入口不显示

## 10. Layouts 页面

`Layouts` 页面分为两个层级：

- 列表页
- 详情页

### 10.1 列表页

列表页展示所有 layout。

每条 layout 的显示文本规则：

- 总是显示显示编号，例如 `layout_1`
- 如果名称非空，则显示为 `layout_1: Name`
- 如果名称为空，则只显示 `layout_1`

列表中的每条 layout 都支持：

- 点击任意区域进入详情页
- 使用右侧入口进入详情页

列表还支持：

- 拖动排序
- 添加 layout

#### 10.1.1 拖动排序

拖动排序会立即改变列表顺序。

排序后：

- 列表展示顺序更新
- 显示编号按新顺序重新计算

当前显示编号是 UI 层编号，不等同于内部稳定标识。

#### 10.1.2 添加 layout

点击 `Add layout` 后：

1. 新建一个 layout
2. 默认名称为空
3. 默认 `Include in cycle = true`
4. 默认网格为 `12 x 6`
5. 默认窗口区域和触发区域使用中心默认值
6. 立即进入这个新 layout 的详情页

### 10.2 详情页

详情页有 3 组逻辑：

- 基本信息
- 预览和编辑
- 底部操作

#### 10.2.1 基本信息组

基本信息组当前包括：

- `Include in cycle`
- `Name`
- `Grid`

##### Include in cycle

决定该 layout 是否参与循环布局。

当此项关闭时：

- 循环布局时跳过此 layout

##### Name

可编辑。

名称为空时：

- 列表页不显示名称部分

##### Grid

用于编辑网格大小。

当前是两项输入：

- 列数
- 行数

当网格大小变化时：

- 窗口区域会按新网格边界裁剪
- 触发区域会按新网格边界裁剪

#### 10.2.2 预览和编辑组

预览和编辑组包含两个 tab：

- `Window overlay`
- `Trigger overlay`

两者共享同一个 layout 草稿，但编辑目标不同。

##### Window overlay

只显示和编辑：

- 窗口区域

不会叠加显示触发区域。

##### Trigger overlay

只显示和编辑：

- 触发区域

不会叠加显示窗口区域。

#### 10.2.3 底部操作

底部操作包含：

- `Delete`
- `Save`

##### Save

只有草稿发生变化时可点击。

点击后：

1. 保存当前草稿
2. 返回 `Layouts` 列表页

##### Delete

删除需要二次确认：

1. 第一次点击进入确认状态
2. 第二次点击执行删除

删除成功后：

1. 删除当前 layout
2. 返回 `Layouts` 列表页

限制：

- 至少保留一个 layout

## 11. Appearance 页面

`Appearance` 页面分为两个区域：

- 顶部固定预览
- 下方配置区

配置区自身可滚动。

顶部预览不随配置区滚动。

### 11.1 页面结构

顶部预览下方有两个 tab：

- `Window overlay`
- `Trigger overlay`

这两个 tab 控制下方显示哪一组配置。

两个 tab 各自都有：

- 一组配置项
- 一个 `Reset to defaults`

### 11.2 Window overlay

这一组用于配置窗口区域预览和运行时窗口高亮相关样式。

当前项包括：

- `Render window area`
- `Fill opacity`
- `Stroke width`
- `Stroke color`

其中：

- `Render window area` 是主开关
- 其余三项从属于这个开关

当主开关关闭时：

- 其余三项显示为禁用状态

### 11.3 Trigger overlay

这一组用于配置触发区域预览和运行时触发区域相关样式。

当前项包括：

- `Render trigger area`
- `Stroke opacity`
- `Grid gap`
- `Stroke color`

其中：

- `Render trigger area` 是主开关
- 其余三项从属于这个开关

当主开关关闭时：

- 其余三项显示为禁用状态

### 11.4 Reset to defaults

两个 tab 各自都有独立的重置动作。

点击后：

- 只重置当前 tab 对应的配置项
- 不影响另一个 tab 的配置

不需要确认。

## 12. Hotkeys 页面

`Hotkeys` 页面不再分组显示，所有快捷键都在一个列表中。

每条快捷键包含 3 个逻辑部分：

- 启用勾选
- 动作
- 快捷键录制

### 12.1 启用勾选

每条快捷键前面的小勾选框控制该条是否启用。

关闭后：

- 该条快捷键不参与实际绑定

### 12.2 动作

动作支持：

- `Apply <layout>`
- `Apply next layout`
- `Apply previous layout`

其中 `Apply <layout>` 依赖当前 layout 列表。

### 12.3 快捷键录制

右侧快捷键区域既负责显示，也负责录制。

录制规则：

- 双击后进入录制
- 输入按键后完成绑定
- `Esc` 清空并退出
- `Backspace` 清空并退出

如果快捷键为空：

- 该条目不绑定任何快捷键

### 12.4 列表操作

列表底部有两个操作：

- 添加
- 删除

#### 添加

点击加号后：

1. 在列表开头插入一条新快捷键
2. 默认启用
3. 默认动作是第一个 layout
4. 默认快捷键为空
5. 自动选中新条目

#### 删除

点击减号后：

- 删除当前选中条目
- 不需要确认

当没有选中项时：

- 删除不可用

## 13. About 页面

`About` 页面当前只有一个信息项：

- `Version`

该值来自应用打包版本号。

## 14. 当前持久化与保存逻辑

当前设置页面的保存方式分两类：

### 14.1 即时保存

以下页面修改后立即保存：

- `General`
- `Appearance`
- `Hotkeys`

### 14.2 草稿后保存

以下页面先编辑草稿，再显式保存：

- `Layouts`

只有点击 `Save` 后，layout 草稿才写回配置。

## 15. 运行时依赖关系

设置界面中的一些项会直接影响运行时行为：

- `Enable`
  - 控制拖动、快捷键、CLI
- `Middle mouse`
  - 控制中键拖动是否启用
- `Modifier + left mouse`
  - 控制修饰键左键拖动是否启用
- `Include in cycle`
  - 控制 layout 是否参与循环
- `Excluded windows`
  - 控制窗口是否允许成为目标
- `Hotkeys`
  - 控制快捷键绑定和启用状态
