# GridMove Settings UI

本文档记录当前 GridMove 设置界面的功能、页面结构和交互行为，用于后续重写 UI 时作为行为参考。

本文档描述的是当前实现的实际行为，不代表最终设计。

## 1. 整体结构

设置窗口采用左右两栏布局：

- 左侧是固定宽度侧边栏，不可折叠。
- 右侧是详情区域。

窗口当前最小尺寸为：

- 宽度 `860`
- 高度 `620`

## 2. 侧边栏

侧边栏顶部显示应用标题：

- `GridMove`

标题下方是 5 个页面入口：

- `General`
- `Layouts`
- `Appearance`
- `Hotkeys`
- `About`

当前侧边栏使用系统 `sidebar` 风格列表，条目包含：

- 图标
- 标题

点击侧边栏条目后：

- 当前页面切换到对应页面
- 会写入页面导航历史

## 3. 顶部标题栏

右侧详情区域顶部有固定标题栏，不跟随页面内容滚动。

标题栏包含：

- 返回按钮
- 前进按钮
- 当前页面标题

### 3.1 页面级导航历史

除 `Layouts` 页的详情态外，前进后退用于页面之间的历史导航。

例如：

1. 进入 `General`
2. 切到 `Appearance`
3. 切到 `Hotkeys`

这时：

- 返回会回到 `Appearance`
- 再返回会回到 `General`
- 前进会按历史重新进入后续页面

### 3.2 Layouts 页特殊导航

`Layouts` 页有两种状态：

- 列表页
- 详情页

当进入 `Layouts` 的详情页时：

- 顶部标题不再显示 `Layouts`
- 改为当前条目的显示编号，例如 `layout_4`

此时：

- 返回按钮会先从详情页回到 `Layouts` 列表页
- 前进按钮可以重新进入刚才那个详情页

## 4. 公共预览组件

多个页面共用同一套预览组件，当前用于：

- `Layouts` 详情页
- `Appearance` 页面

公共特征如下：

- 预览比例固定，不随窗口宽高自由拉伸
- 预览配色固定，不跟随系统浅色或深色模式变化
- 预览内部有菜单栏区域
- 菜单栏内的苹果图标和时间会随菜单栏高度缩放
- 预览中的屏幕网格与菜单栏网格使用统一的绘制样式

当前预览配色采用固定灰色系背景和固定网格底色，目的是保证可读性。

## 5. General 页面

`General` 页面使用系统 `Form` 风格，主要分为 3 组内容：

- 总开关
- 鼠标触发相关设置
- 排除窗口列表

### 5.1 Enable

页面顶部有总开关：

- 标题：`Enable`
- 说明：允许拖动触发、布局快捷键和命令行布局动作

当该开关关闭时：

- 拖动功能不可用
- 快捷键不可用
- CLI 动作不可用

### 5.2 Press And Drag

此部分受总开关控制。

当总开关关闭时：

- 这一组整体禁用

包含两个触发入口：

- `Middle Mouse`
- `Modifier + Left Mouse`

#### 5.2.1 Middle Mouse

控制中键长按触发拖动网格。

#### 5.2.2 Modifier + Left Mouse

控制修饰键加左键触发拖动网格。

下方有修饰键组合列表：

- 使用列表显示所有修饰键组合
- 当前列表支持选中某一项

底部有加减按钮：

- `+`：打开添加修饰键组合弹窗
- `-`：删除当前选中的修饰键组合

当没有选中任何条目时：

- 减号不显示

添加修饰键组合弹窗包含：

- `Cmd`
- `Ctrl`
- `Shift`
- `Option`

用户勾选后点击 `Add` 才会添加。

### 5.3 Excluded Windows

用于维护排除窗口规则。

当前以表格形式展示：

- `Value`
- `Type`

支持两种类型：

- `Bundle ID`
- `Window Title`

底部有加减按钮：

- `+`：打开添加排除项弹窗
- `-`：删除当前选中项

当没有选中任何条目时：

- 减号不显示

添加排除项弹窗包含：

- 类型选择
- 文本输入框

类型选择项：

- `Bundle ID`
- `Window Title`

## 6. Layouts 页面

`Layouts` 页面当前分为两个状态：

- 列表页
- 详情页

### 6.1 列表页

列表页展示所有 layout 条目。

每个条目显示：

- `layout_序号`
- 如果有名称，则显示 `layout_序号: 名称`
- 如果名称为空，则只显示 `layout_序号`

例如：

- `layout_1: Left 1/3`
- `layout_12`

每个条目右侧有箭头按钮。

交互规则：

- 点击条目任意位置，进入详情页
- 点击右侧箭头，也进入详情页

#### 6.1.1 拖动排序

列表支持拖动排序。

排序后：

- 列表展示顺序立即更新
- `layout_序号` 的显示编号也立即按新顺序重算

当前这里更新的是显示编号，不是内部持久化的原始 `id` 字符串。

#### 6.1.2 Add a layout

列表底部有按钮：

- `Add a layout`

点击后：

1. 新建一个 layout
2. 默认名称为空
3. 默认网格是 `12 x 6`
4. 默认窗口区域是中间区域
5. 默认触发区域与窗口区域相同
6. 直接进入该 layout 的详情页

### 6.2 详情页

详情页顶部标题为当前条目的显示编号，例如：

- `layout_1`
- `layout_11`

详情页当前包含 3 部分：

- 基本信息
- 预览编辑区域
- 底部操作按钮

#### 6.2.1 基本信息

基本信息区域包含：

- `Include in cycle`
- `Name`
- `Grid`

##### Include in cycle

控制该 layout 是否参与循环布局。

##### Name

可编辑文本。

名称为空时：

- 列表页只显示 `layout_序号`

##### Grid

当前支持分别编辑：

- 列数
- 行数

界面展示为：

- 左边 `Grid`
- 中间列输入框
- 一个 `x`
- 右边行输入框

输入时会做最小值限制：

- 列数最少为 `1`
- 行数最少为 `1`

当网格大小变化后：

- 窗口区域和触发区域会按当前规则裁剪到新网格范围内

#### 6.2.2 预览编辑区域

预览编辑区域位于基本信息下方。

顶部有两个 tab：

- `Window overlay`
- `Trigger overlay`

##### Window overlay

此状态下：

- 只显示窗口区域预览
- 只编辑窗口区域
- 不显示触发区域叠加

##### Trigger overlay

此状态下：

- 只显示触发区域预览
- 只编辑触发区域
- 不显示窗口区域叠加

这两个编辑器都复用公共预览组件的固定比例和固定配色。

#### 6.2.3 Save

底部有 `Save` 按钮。

按钮行为：

- 只有当前草稿有修改时才可点击
- 点击后保存当前 layout
- 保存成功后返回 `Layouts` 列表页

#### 6.2.4 Delete

底部有 `Delete` 按钮。

删除采用二次确认：

1. 第一次点击，按钮文字变成 `Confirm delete`
2. 第二次点击，执行删除

删除成功后：

- 返回 `Layouts` 列表页

限制：

- 至少保留一个 layout

如果当前只剩一个 layout：

- 不允许删除

## 7. Appearance 页面

`Appearance` 页面使用固定预览加下方滚动配置的结构。

整体结构：

- 顶部固定预览
- 下方 tab
- tab 下方是一组配置
- 配置区域下方有 `Reset to defaults`

### 7.1 预览区域

预览区域位于标题栏下方，不会随着下方配置滚动。

当前预览固定使用：

- `12 x 6` 网格
- 固定示例窗口区域
- 固定示例触发区域

预览会实时反映用户对样式配置的修改。

### 7.2 标签页

当前有两个 tab：

- `Window overlay`
- `Trigger overlay`

两个 tab 铺满整行宽度。

### 7.3 Window overlay

用于配置窗口预览样式。

当前包含：

- `Render window area`
- `Fill opacity`
- `Stroke width`
- `Stroke color`

当 `Render window area` 关闭时：

- 下方相关项显示为禁用态

### 7.4 Trigger overlay

用于配置触发区域预览样式。

当前包含：

- `Render trigger area`
- `Stroke opacity`
- `Grid gap`
- `Stroke color`

当 `Render trigger area` 关闭时：

- 下方相关项显示为禁用态

### 7.5 Reset to defaults

两个 tab 下方都显示：

- `Reset to defaults`

点击后：

- 当前 tab 对应的样式配置恢复为程序默认值

不需要二次确认。

## 8. Hotkeys 页面

`Hotkeys` 页面当前使用单列表结构，不再分组显示。

顶部有一条说明文字，说明双击快捷键可以重新设置。

### 8.1 列表内容

每一条快捷键包含 3 部分：

- 左侧启用勾选
- 中间动作
- 右侧快捷键录制区域

#### 8.1.1 启用勾选

小勾选框用于控制该条快捷键是否启用。

关闭后：

- 该条快捷键不会触发运行时行为

#### 8.1.2 动作

当前支持的动作有：

- `Switch to <layout>`
- `Switch to next layout`
- `Switch to previous layout`

其中 `Switch to <layout>` 的布局名称使用当前 layout 的显示文案。

#### 8.1.3 快捷键显示和录制

右侧快捷键区域会显示系统风格的符号组合，例如：

- `⌃⌘⇧A`
- `⌥⌘L`

录制方式：

- 双击右侧快捷键区域，进入录制状态
- 输入新快捷键后完成绑定
- 按 `Esc` 清空并退出
- 按 `Backspace` 清空并退出

如果为空快捷键：

- 该条目不会绑定任何快捷键

### 8.2 列表底部按钮

列表底部左下角有：

- `+`
- `-`

#### 加号

点击后：

1. 在列表开头插入一个新条目
2. 默认启用
3. 默认快捷键为空
4. 默认动作是第一个 layout
5. 自动选中新条目

#### 减号

点击后：

- 删除当前选中的条目
- 不需要确认

当没有选中任何条目时：

- 减号不可用

## 9. About 页面

`About` 页面当前只有一个信息项：

- `Version`

版本值读取自应用打包信息中的：

- `CFBundleShortVersionString`

如果运行环境中无法读取该值，则回退到默认版本字符串。

## 10. 当前实现中的重要补充说明

### 10.1 Enable 的影响范围

`General` 页面中的 `Enable` 是全局总开关。

关闭后会影响：

- 拖动触发
- 快捷键
- CLI 动作

### 10.2 Layout 的显示编号与内部 ID

当前 `Layouts` 页面显示的是按顺序生成的显示编号：

- `layout_1`
- `layout_2`
- `layout_3`

这套编号会随着拖动排序即时变化。

当前内部真正用于数据绑定和快捷键动作的 `LayoutPreset.id` 仍然保持原始值，不会因为排序自动改写。

### 10.3 页面状态与保存行为

当前设置窗口没有统一的全局保存按钮。

不同页面的保存方式如下：

- `General`：修改即保存
- `Appearance`：修改即保存
- `Hotkeys`：修改即保存
- `Layouts`：编辑草稿，点击 `Save` 后保存
