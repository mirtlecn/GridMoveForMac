# GridMove UI Logic

本文档记录当前主要 UI 的功能逻辑、页面结构、分组关系、状态依赖和交互行为。

本文档不记录视觉样式、间距、字号、最小尺寸等表现层细节。

## 1. UI 范围

当前 UI 范围包括：

- 菜单栏菜单
- 权限引导窗口
- 设置窗口

## 2. 文案规则

当前可见 UI 文案遵循以下规则：

- 默认使用 sentence case
- 只有首单词首字母大写
- 缩写和专有名词按实际需要保留大写，例如 `Bundle ID`、`GridMove`

当前可见 UI 文案统一集中在：

- `Sources/GridMove/Support/UICopy.swift`

该文件用于：

- 统一菜单栏文案
- 统一权限引导窗口文案
- 统一设置窗口文案
- 为后续翻译提供单一入口

## 3. 菜单栏菜单

菜单栏菜单包含以下固定结构：

- `Enable`
- 分隔线
- 布局动作项
- 分隔线
- `Settings…`
- 分隔线
- `Quit`

### 3.1 Enable

`Enable` 是全局开关的菜单入口。

点击后：

- 切换应用启用状态
- 同步更新设置页中的全局启用状态
- 同步影响拖动触发、快捷键和 CLI 动作

### 3.2 布局动作项

布局动作项由当前配置动态生成，顺序为：

- `Apply previous layout`
- `Apply next layout`
- 每个 layout 一项 `Apply <layout name>`

行为规则：

- 如果该动作配置了快捷键，菜单右侧显示第一个已启用快捷键
- 点击菜单项时，操作当前解析到的目标窗口
- 菜单项本身不维护独立的布局状态，只触发现有布局应用逻辑

### 3.3 Settings

点击 `Settings…` 后：

- 打开设置窗口
- 如果窗口已存在，则激活并显示已有窗口

### 3.4 Quit

点击 `Quit` 后：

- 退出应用

## 4. 权限引导窗口

当应用检测到缺少辅助功能权限时，会显示权限引导窗口。

当前窗口行为：

- 激活应用
- 显示引导窗口
- 不使用全局置顶层级
- 不强制压过系统设置自己的弹窗

窗口包含：

- 标题 `Accessibility access is required`
- 说明文案
- `Request accessibility access`
- `Open accessibility settings`

### 4.1 Request accessibility access

点击后：

- 触发系统辅助功能权限请求

### 4.2 Open accessibility settings

点击后：

- 打开系统设置中的辅助功能权限页面

## 5. 设置窗口总体结构

当前设置窗口使用标准 macOS 标题栏。

设置内容区由两部分组成：

- 顶部 tabs
- 下方单列页面内容

当前阶段是 settings v2 的视觉原型。

行为规则：

- 不使用左侧侧边栏
- 不使用工具栏内的页面导航
- 页面主体使用固定宽度单列内容
- tabs 切换只切换页面内容

## 6. 页面列表

当前设置界面包含 5 个页面：

- `General`
- `Layouts`
- `Appearance`
- `Hotkeys`
- `About`

点击顶部 tabs 后：

- 当前页面切换到对应页面

当前原型阶段的页面状态说明：

- `General`：展示静态视觉原型
- `Layouts`：占位页
- `Appearance`：占位页
- `Hotkeys`：占位页
- `About`：占位页

## 7. 工具栏导航逻辑

当前设置窗口不包含页面级工具栏导航。

因此当前不存在以下行为：

- 返回/前进
- 页面历史
- `Layouts` 内部详情导航

## 8. 共享预览组件逻辑

多个页面复用同一套预览组件逻辑，目前包括：

- `Layouts`
- `Appearance`

共享预览组件的逻辑要求：

- 使用统一的数据结构描述窗口区域和触发区域
- 支持屏幕区域和菜单栏区域
- 菜单栏区域与屏幕区域共用同一套命中和显示逻辑
- 预览组件在不同页面中应保持同一套行为

当前原型阶段：

- 这两页还未接回真实预览逻辑
- 共享预览组件要求保持不变，供后续完整迁移使用

## 9. General 页面

当前 `General` 页面用于预览新的 settings v2 视觉结构。

页面包含 3 组内容：

- 总开关
- 鼠标触发设置
- 排除窗口设置

### 9.1 总开关

页面顶部展示全局开关卡片：

- `Enable`

当前原型阶段：

- 使用真实文案
- 使用本地临时状态
- 不写回真实配置

### 9.2 鼠标触发设置

此组包含两个独立开关：

- `Middle mouse`
- `Modifier + left mouse`

当前原型阶段，这两项只用于展示视觉层级和间距关系。

#### 9.2.1 Middle mouse

当前用于展示独立开关行样式。

#### 9.2.2 Modifier + left mouse

当前用于展示带附属列表的卡片结构。

其下有一个附属列表，用于维护修饰键组合。

修饰键组合列表支持：

- 选择一项
- 添加
- 删除

当前原型阶段：

- 仅展示列表、选中态和按钮排版
- 不打开真实弹窗
- 不执行真实增删逻辑

### 9.3 排除窗口设置

此组用于展示排除规则卡片。

每条规则有两个字段：

- 值
- 类型

类型支持：

- `Bundle ID`
- `Window title`

排除规则列表支持：

- 选择一项
- 添加
- 删除

当前原型阶段：

- 仅展示列表、选中态和按钮排版
- 不打开真实弹窗
- 不执行真实增删逻辑

## 10. Layouts 页面

当前 `Layouts` 页面是占位页。

用于表达后续迁移方向：

- 将改成更简单的列表入口
- 新增和编辑将改为 sheet
- 当前阶段不展示真实列表和编辑器

## 11. Appearance 页面

当前 `Appearance` 页面是占位页。

用于表达后续迁移方向：

- 仍然保留预览优先结构
- 顶部保留独立 tab 切换
- 下方使用同一套卡片视觉语言
- 当前阶段不接真实配置项和重置逻辑

## 12. Hotkeys 页面

当前 `Hotkeys` 页面是占位页。

后续会迁移到新的卡片视觉，但保留原有快捷键模型和录制控件。

## 13. About 页面

当前 `About` 页面是占位页。

## 14. 当前持久化与保存逻辑

当前原型阶段不接回真实持久化。

这意味着：

- `General` 的本地状态不会写回配置
- 占位页不执行真实保存
- 原有持久化模型保持不变，供后续完整迁移接回

## 15. 运行时依赖关系

当前原型阶段，设置窗口不直接影响运行时行为。

原有运行时依赖关系保留在逻辑层中，但尚未重新接入新的 settings v2 界面。
